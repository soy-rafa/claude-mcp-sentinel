# MCP Sentinel — reproducible benchmark

Security tools should be judged on evidence, not claims. Everything here you can
reproduce locally in seconds. All attack data is **inert** (it describes what an
attack would do, as data; nothing executes) and ships base64-at-rest.

## Headline numbers

| Suite | Command | Result |
|-------|---------|--------|
| Unit + regression | `python3 tests/test_hook.py` | 114/114 passed |
| Precision (false-positive gate) | `python3 tests/precision_check.py` | **FP = 0 / 91**, recall 44/44 |
| Red-team (all batteries) | `python3 tests/redteam_check.py` | **69 attacks, 0 missed, 0 false positives** |

The red-team runner aggregates every `tests/fixtures/redteam_scenarios*.json`:
- **Battery 1** — 39 adversarially-designed scenarios (single + combined vectors).
- **Battery 2** — 35 scenarios recreating **real documented incidents**: the Postmark
  MCP backdoor, tool poisoning / "line jumping" (Invariant Labs / Trail of Bits),
  SKILL.md prompt injection, credential exfiltration, persistence & RCE
  (CVE-2025-59536, CVE-2025-66032 bypasses), cross-tool injection + cloud IMDS, and a
  supply-chain set (typosquat MCP, `*_BASE_URL` LLM-traffic hijack, trojaned
  postinstall). Each battery includes benign look-alike decoys to catch over-blocking.

## Methodology (why the number is meaningful)

- **Per-layer verification.** Each attack must be caught by the layer that *should*
  catch it (decide / attack-chain / config-scan / dataflow), not by an incidental
  match. 0 partial seams.
- **False-positive gate.** Precision is a first-class metric: 91 benign developer
  actions must pass untouched. A detection that raises FP is a regression.
- **Adversarial design.** Scenarios were generated by independent agents told to
  emulate the real attack faithfully, not to pass. Gaps found this way (7 in battery
  1) were fixed and locked as regressions.
- **Inert + AV-safe.** No scenario executes anything; fixtures are base64-wrapped so
  a host antivirus does not flag the corpus.

## Reproduce it

```
git clone <repo> && cd mcp-sentinel
python3 tests/test_hook.py
python3 tests/precision_check.py
python3 tests/redteam_check.py
```

No install, no network, no API key required.

## Contribute an attack

The most valuable contribution is a **bypass**: a tool call or config that slips
through. Add it (inert) to a `redteam_scenarios*.json` fixture or send it privately
(see `SECURITY.md`). If Sentinel misses it, that is a bug we want.
